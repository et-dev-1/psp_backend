# Backend: Database Migrations & Schema Changes Skill

## When to Use
- User asks to add a new database table or column
- Modifying existing table structure
- Creating indexes or relationships
- Running database initialization scripts

## Database Files

| File | Purpose |
|------|---------|
| `schema.sql` | Current database schema (source of truth) |
| `bootstrap.sql` | Initial database structure (for first setup) |
| `scripts/apply-schema.ts` | Migration runner (updates schema incrementally) |
| `db.d.ts` | TypeScript type definitions for tables |

## Adding a New Table

### Step 1: Add to schema.sql
```sql
-- In schema.sql, add your table definition

CREATE TABLE IF NOT EXISTS products (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  price DECIMAL(10, 2) NOT NULL,
  stock INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  INDEX idx_user_id (user_id),
  INDEX idx_created_at (created_at)
);
```

### Step 2: Update db.d.ts with Types
```typescript
// In db.d.ts

export interface Product {
  id: number;
  user_id: number;
  name: string;
  description: string | null;
  price: number;
  stock: number;
  created_at: Date;
  updated_at: Date;
}

export interface Database {
  products: Product;
  // ... other tables
}
```

### Step 3: Run Migration
```bash
npm run db:init
```

This executes `scripts/apply-schema.ts` which:
1. Connects to the database
2. Reads `schema.sql`
3. Applies any new tables/columns/indexes
4. Logs changes made

## Adding a Column to Existing Table

### Step 1: Modify schema.sql
```sql
-- In schema.sql, update the existing table

CREATE TABLE IF NOT EXISTS orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  status ENUM('pending', 'confirmed', 'shipped', 'delivered') DEFAULT 'pending',
  total_amount DECIMAL(10, 2) NOT NULL,
  -- NEW COLUMN BELOW
  tracking_number VARCHAR(255),
  -- NEW COLUMN ABOVE
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### Step 2: Update db.d.ts
```typescript
export interface Order {
  id: number;
  user_id: number;
  status: 'pending' | 'confirmed' | 'shipped' | 'delivered';
  total_amount: number;
  tracking_number?: string;  // NEW PROPERTY
  created_at: Date;
  updated_at: Date;
}
```

### Step 3: Run Migration
```bash
npm run db:init
```

## Custom Migrations (If Needed)

For complex migrations, create a new file in `scripts/`:

```typescript
// scripts/add-product-variants.ts

import db from '../db';
import { execute } from '../db';

async function migrate() {
  try {
    const connection = await db.getConnection();

    // Create new table
    await connection.query(`
      CREATE TABLE IF NOT EXISTS product_variants (
        id INT PRIMARY KEY AUTO_INCREMENT,
        product_id INT NOT NULL,
        size VARCHAR(50),
        color VARCHAR(50),
        sku VARCHAR(100),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (product_id) REFERENCES products(id)
      )
    `);

    // Add column to existing table
    await connection.query(`
      ALTER TABLE products 
      ADD COLUMN IF NOT EXISTS has_variants BOOLEAN DEFAULT FALSE
    `);

    // Update existing data if needed
    await connection.query(`
      UPDATE products SET has_variants = TRUE 
      WHERE id IN (SELECT DISTINCT product_id FROM product_variants)
    `);

    connection.release();
    console.log('Migration: product variants completed');
  } catch (error) {
    console.error('Migration failed:', error);
    throw error;
  }
}

migrate().catch(console.error);
```

Run with: `ts-node scripts/add-product-variants.ts`

## Common Schema Patterns

### Users Table
```sql
CREATE TABLE IF NOT EXISTS users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  role ENUM('admin', 'seller', 'customer') DEFAULT 'customer',
  is_verified BOOLEAN DEFAULT FALSE,
  two_factor_enabled BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_email (email)
);
```

### Orders Table
```sql
CREATE TABLE IF NOT EXISTS orders (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  order_number VARCHAR(50) UNIQUE,
  status ENUM('pending', 'confirmed', 'shipped', 'delivered', 'cancelled') DEFAULT 'pending',
  total_amount DECIMAL(10, 2) NOT NULL,
  shipping_address TEXT,
  payment_method VARCHAR(50),
  payment_status ENUM('unpaid', 'paid', 'refunded') DEFAULT 'unpaid',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  INDEX idx_user_id (user_id),
  INDEX idx_status (status)
);
```

### Products Table
```sql
CREATE TABLE IF NOT EXISTS products (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  price DECIMAL(10, 2) NOT NULL,
  stock INT DEFAULT 0,
  image_url VARCHAR(500),
  category VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  INDEX idx_user_id (user_id),
  INDEX idx_category (category)
);
```

## Database Connection Pattern

```typescript
// Always use connection pooling from db.ts
const connection = await db.getConnection();

try {
  const [rows] = await connection.query('SELECT * FROM table');
  connection.release();
  return rows;
} catch (error) {
  connection.release();
  throw error;
}
```

## Index Best Practices
✓ Add indexes on frequently queried columns (user_id, email, status)  
✓ Index foreign keys  
✓ Index timestamp columns for sorting/filtering  
✗ Don't over-index; limit to 3-4 indexes per table  
✗ Avoid indexes on low-cardinality columns (boolean, enum with few values)  

## Testing Schema Changes

After running `npm run db:init`:
1. Connect to MySQL: `mysql -u root -p`
2. Use database: `USE your_database_name;`
3. Check table: `DESCRIBE products;`
4. Verify indexes: `SHOW INDEXES FROM products;`

## Workflow

**For Adding a Table:**
1. Write CREATE TABLE in schema.sql
2. Add TypeScript interface in db.d.ts
3. Run `npm run db:init`
4. Create API routes using the new table
5. Add WebSocket events if needed

**For Adding a Column:**
1. Add ALTER TABLE in schema.sql
2. Update interface in db.d.ts
3. Run `npm run db:init`
4. Update existing routes if needed

**For Renaming/Deleting (High Risk):**
1. Create a custom migration in scripts/
2. Test thoroughly before running in production
3. Backup database first
4. Run migration script
5. Update schema.sql and db.d.ts

## Common Gotchas
✗ Forgetting to release database connections  
✗ Adding columns without DEFAULT values (breaks existing code)  
✗ Creating foreign keys that reference non-existent tables  
✗ Not updating db.d.ts after schema changes  
✓ Always backup database before large migrations  
✓ Test schema changes in dev first  
✓ Use CONSTRAINT names for clarity
