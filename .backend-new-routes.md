# Backend: Adding New API Routes Skill

## When to Use
- User asks to create a new API endpoint
- Adding CRUD operations for a resource
- Implementing business logic handlers
- Integrating authentication/authorization

## Route Structure

### Basic Route Pattern
```typescript
// In server.ts

app.post('/api/resource', authenticateToken, async (req, res) => {
  try {
    const userId = req.user?.id;
    const { field1, field2 } = req.body;

    // Validation
    if (!field1 || !field2) {
      return res.status(400).json({ error: 'Missing required fields' });
    }

    // Database query
    const connection = await db.getConnection();
    const [result] = await connection.query(
      'INSERT INTO resources (user_id, field1, field2) VALUES (?, ?, ?)',
      [userId, field1, field2]
    );
    connection.release();

    // Response
    res.status(201).json({
      id: result.insertId,
      field1,
      field2,
      createdAt: new Date()
    });
  } catch (error) {
    console.error('Error creating resource:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

## Common Route Patterns

### GET (Read)
```typescript
// Get all resources (with pagination)
app.get('/api/resources', authenticateToken, async (req, res) => {
  try {
    const userId = req.user?.id;
    const page = Math.max(1, parseInt(req.query.page as string) || 1);
    const limit = Math.min(100, parseInt(req.query.limit as string) || 10);
    const offset = (page - 1) * limit;

    const connection = await db.getConnection();
    const [resources] = await connection.query(
      'SELECT * FROM resources WHERE user_id = ? LIMIT ? OFFSET ?',
      [userId, limit, offset]
    );
    const [[{ total }]] = await connection.query(
      'SELECT COUNT(*) as total FROM resources WHERE user_id = ?',
      [userId]
    );
    connection.release();

    res.json({
      data: resources,
      pagination: { page, limit, total }
    });
  } catch (error) {
    console.error('Error fetching resources:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});

// Get single resource by ID
app.get('/api/resources/:id', authenticateToken, async (req, res) => {
  try {
    const { id } = req.params;
    const userId = req.user?.id;

    const connection = await db.getConnection();
    const [rows] = await connection.query(
      'SELECT * FROM resources WHERE id = ? AND user_id = ?',
      [id, userId]
    );
    connection.release();

    if (!rows.length) {
      return res.status(404).json({ error: 'Resource not found' });
    }

    res.json(rows[0]);
  } catch (error) {
    console.error('Error fetching resource:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

### POST (Create)
```typescript
app.post('/api/resources', authenticateToken, async (req, res) => {
  try {
    const userId = req.user?.id;
    const { name, description, price } = req.body;

    // Validate
    if (!name || !description) {
      return res.status(400).json({
        error: 'Missing required fields: name, description'
      });
    }

    // Database insert
    const connection = await db.getConnection();
    const [result] = await connection.query(
      'INSERT INTO resources (user_id, name, description, price) VALUES (?, ?, ?, ?)',
      [userId, name, description, price || 0]
    );
    connection.release();

    res.status(201).json({
      id: result.insertId,
      name,
      description,
      price: price || 0
    });
  } catch (error) {
    console.error('Error creating resource:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

### PUT/PATCH (Update)
```typescript
app.put('/api/resources/:id', authenticateToken, async (req, res) => {
  try {
    const { id } = req.params;
    const userId = req.user?.id;
    const { name, description, price } = req.body;

    // Verify ownership
    const connection = await db.getConnection();
    const [current] = await connection.query(
      'SELECT * FROM resources WHERE id = ? AND user_id = ?',
      [id, userId]
    );

    if (!current.length) {
      connection.release();
      return res.status(404).json({ error: 'Resource not found' });
    }

    // Update
    await connection.query(
      'UPDATE resources SET name = ?, description = ?, price = ? WHERE id = ?',
      [name || current[0].name, description || current[0].description, price !== undefined ? price : current[0].price, id]
    );
    connection.release();

    res.json({ id, name, description, price });
  } catch (error) {
    console.error('Error updating resource:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

### DELETE (Remove)
```typescript
app.delete('/api/resources/:id', authenticateToken, async (req, res) => {
  try {
    const { id } = req.params;
    const userId = req.user?.id;

    // Verify ownership before delete
    const connection = await db.getConnection();
    const [rows] = await connection.query(
      'SELECT * FROM resources WHERE id = ? AND user_id = ?',
      [id, userId]
    );

    if (!rows.length) {
      connection.release();
      return res.status(404).json({ error: 'Resource not found' });
    }

    // Delete
    await connection.query('DELETE FROM resources WHERE id = ?', [id]);
    connection.release();

    res.json({ message: 'Resource deleted successfully' });
  } catch (error) {
    console.error('Error deleting resource:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

## Authentication Middleware

### Using JWT Auth
```typescript
// Middleware included in all protected routes
app.post('/api/protected', authenticateToken, async (req, res) => {
  // req.user is automatically set by middleware
  const userId = req.user?.id;
  const userRole = req.user?.role;
});
```

### Role-Based Access
```typescript
// Admin-only route
app.get('/api/admin/users', authenticateToken, async (req, res) => {
  if (req.user?.role !== 'admin') {
    return res.status(403).json({ error: 'Forbidden: Admin access required' });
  }
  // Route logic
});

// Seller-only route
app.post('/api/seller/products', authenticateToken, async (req, res) => {
  if (req.user?.role !== 'seller') {
    return res.status(403).json({ error: 'Forbidden: Seller access required' });
  }
  // Route logic
});
```

## Database Connection Pattern

```typescript
// Get connection from pool
const connection = await db.getConnection();

try {
  // Execute queries
  const [result] = await connection.query('SELECT * FROM table');
  
  // Insert/Update
  const [insertResult] = await connection.query(
    'INSERT INTO table (col1, col2) VALUES (?, ?)',
    [val1, val2]
  );
  
  // Don't forget to release!
  connection.release();
  
  return result;
} catch (error) {
  connection.release();
  throw error;
}
```

## WebSocket Events (server.ts)

After updating a resource, notify connected clients:
```typescript
// After updating order status
io.to(`order:${orderId}`).emit('order-updated', {
  orderId,
  status: newStatus,
  updatedAt: new Date()
});

// Broadcast product change to all subscribers
io.to(`product:${productId}`).emit('product-changed', {
  productId,
  name,
  price,

});
```

## File Locations
All routes go in: **server.ts** (inline, not in separate files)

Group by feature:
1. Authentication routes (signin, signup, 2fa)
2. User routes (profile, settings)
3. Product routes (CRUD operations)
4. Order routes (create, list, update status)
5. Payment routes (process payment, webhooks)
6. Admin routes (user management, analytics)

## Common Gotchas
✗ Forgetting to release database connections  
✗ Not verifying user ownership before update/delete  
✗ Returning sensitive data (passwords, tokens)  
✗ Not handling connection timeouts  
✗ Missing error handling in try-catch  
✓ Always check user role before admin operations  
✓ Always validate required fields  
✓ Use parameterized queries to prevent SQL injection  
✓ Release connections even on errors
